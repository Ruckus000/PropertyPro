# Spec: P1-28 — Email Infrastructure

> Configure Resend for transactional email delivery with proper DNS records for deliverability.

## Phase
1 — Compliance Core

## Priority
P1

## Dependencies
- P0-00

## Functional Requirements
- Set up Resend account with dedicated sending domain (mail.propertyprofl.com)
- Configure SPF, DKIM, DMARC DNS records
- Build React Email templates in packages/email: invitation, password reset, meeting notice, compliance alert, general announcement
- Use consistent from address
- Include List-Unsubscribe header on non-critical emails
- Test delivery to Gmail, Outlook, Yahoo

## Acceptance Criteria
- [ ] Email templates render correctly with React Email
- [ ] Emails sent via Resend arrive in inbox (not spam) for Gmail
- [ ] SPF/DKIM/DMARC records configured
- [ ] List-Unsubscribe header present
- [ ] `pnpm test` passes for template rendering

## Technical Notes
- From address: noreply@mail.propertyprofl.com or support@mail.propertyprofl.com
- SPF: include:sendgrid.net (if using Resend's infrastructure)
- DKIM: auto-generated by Resend, add CNAME records
- DMARC: p=quarantine (strict without p=reject until monitoring stable)
- React Email: @react-email/components for templating
- Test with mail.tm or similar for spam testing
